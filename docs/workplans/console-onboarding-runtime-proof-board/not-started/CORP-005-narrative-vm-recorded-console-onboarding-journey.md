# CORP-005 Narrative VM-Recorded Console Onboarding Journey

## Goal

Replace the broad synthetic page sweep as the primary review artifact with one
narrative VM-recorded Console onboarding and runtime-backed data journey.

## Narrative

The primary recording should show:

1. fresh Console open
2. adapter setup through UI
3. connection state and post-connect controls
4. backfill or sync progression
5. runtime-backed records, contacts, and channels

## Acceptance

- one `full-session.webm` is the primary review artifact
- the recording shows the product journey a human reviewer actually cares about
- debug traces and per-test videos remain secondary media
