# Golden Journey Validation Board

## Purpose

This board executes the next review layer on top of the cleanroom model:

- whole-sandbox session recording
- one primary demo artifact per ticket or feature lane
- explicit human-shaped validation scripts before execution
- a small set of flagship narrative flows that grow into the cumulative golden
  journey
- Dispatch as the long-term board, run, and review surface

This lane is not about inventing a second validation substrate.

It is about making the existing sandbox proof model reviewable, human-shaped,
and product-native.

## Scope

In scope:

- record the whole sandbox session from startup through proof completion
- attach one canonical primary demo artifact to the owning run
- surface exact validation words and actions before execution
- define retention rules for success versus failure
- build the first 2-3 flagship narrative flows
- move review toward Dispatch-owned boards and run detail

Out of scope:

- replacing structured proof bundles with video-only signoff
- removing targeted coverage tests
- making hosted Frontdoor flows the required default path
- using Playwright-only browser capture as the long-term primary recording
