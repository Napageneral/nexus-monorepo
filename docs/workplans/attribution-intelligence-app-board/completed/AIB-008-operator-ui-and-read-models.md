# AIB-008 Operator UI And Read Models

## Goal

Build the first operator-facing UI and read models for connected inputs,
freshness, paid performance, website funnel, backend outcomes, and row-level
inspection.

## Acceptance

1. the UI reads app-owned marts and facts rather than adapter payloads
2. operators can inspect connected inputs and freshness clearly
3. operators can inspect paid performance, website funnel, and backend outcomes
4. one row-level inspector exposes attribution evidence for debugging
