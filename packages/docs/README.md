# Package Docs

Start here to understand the Nex package system.

Recommended reading order:

1. `PACKAGE_SYSTEM.md` - package model, hard rules, release/publication pipeline
2. `PACKAGE_STANDALONE_REPO_MODEL.md` - repo-boundary rule for all package roots
3. `PACKAGE_INDEX.md` - current package-family navigation and status notes
4. `PACKAGE_PUBLISH_AND_SKILL_CUTOVER.md` - hard-cutover path for canonical publish flow and attached-skill enforcement
5. `PACKAGE_SHARED_PUBLISH_WORKPLAN.md` - implementation plan for the shared package publish wrapper
6. `PACKAGE_ENFORCEMENT.md` - manifest-root discovery and enforcement model
7. `APP_FAMILY_NORMALIZATION.md` - app family container vs package-root rules
