Website Input Sender Demo

Normal website, real browser, explicit first-party events.

This page lets us exercise the browser sender on a public site and point it at a Nex runtime collector. It stores local config in the browser and emits canonical events through website-input.collect.

Collector Config

Use a local runtime URL for dogfood, like http://127.0.0.1:18789. If this page is served from Vercel, that origin must be allowed by your runtime CORS policy.

Expected Flow

1. Create a website-input installation in Nex.
2. Paste the installation id and sender token here.
3. Click a CTA or handoff below.
4. Watch the response payload and local event snapshot.
