# WIB-007 Workplan

## Objective

Define the companion pixel ownership policy that sits alongside the first-party
website-input sender while forcing one explicit owner path per signal.

## Execution Steps

1. Define the one-owner rule for each companion signal.
2. Define the ownership decision tree for custom code, GTM, and native
   platform integrations.
3. Define the duplicate-ownership failure conditions.
4. Define the Wix-specific review rules for Meta, Google, and TikTok.
5. Define how the policy is checked during the install proof.

## Deliverables

- ownership decision matrix
- duplicate-ownership failure rules
- Wix review checklist
- install-note template for the chosen owner path

## Exit Condition

This workplan is complete when an operator can state who owns each companion
pixel, where it fires, and how duplication is prevented.
